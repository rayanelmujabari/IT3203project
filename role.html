What does HTTP do?

HTTP is stateless by default, each request is independent, however tools like cookies help gather data that may be useful (Barth, 2011). It also supports secure connections between the client and the server. Data is encrypted using TLS (Transport Layer Security) / SSL (Secure Sockets Layer) preventing any issues during transit. Encryption and Authentication are both key structures in TLS (Transport Layer Security) / SSL (Secure Sockets Layer) protocol (Developers, HTTP: MDN). HTTP also provides mechanisms for identifying and retrieving web resources through URI (Uniform Resource Identifiers) (Berners-Lee et al., 2005).
